# UI Design Uplift — Tesla × Apple

> Generated from the `hem-ui-design-uplift` multi-agent design workflow (8 agents).
> Implementation reference for the 6-phase roadmap. Accuracy guardrails are binding.

## Design language

A near-black, instrument-cluster calm where one number per surface carries the weight and everything else recedes to quiet monochrome — Tesla's live energy-flow as the only thing that visibly moves, Apple's dramatic typographic contrast and meaning-only colour as the discipline. Chrome disappears: cards float on depth, not borders; colour appears only when it encodes a real-world state (PV yellow, battery green, import red, the single blue accent for interaction). Restraint is the brand.

**Unified (post-synthesis):**

A near-black instrument-cluster calm: one focal number per surface carries the weight, everything else recedes to quiet monochrome. CHROME: cards float on depth (two-layer shadow), never borders or per-card accent edges/tints — the deprecated .widget-accent bar and --widget-tint corner gradient are deleted; all WidgetTone values map to one flat --bg-card surface (prop kept one release for compile-safety, paints nothing). --radius-lg = 16px. COLOUR is monochrome-first and spent ONLY to encode a real-world state: energy-domain colours (--pv yellow, --batt/--export green, --grid blue, --house purple, --import red) appear only on the live power-flow, on domain charts (cost bars, energy stack, legends), and on the single value that IS that domain (SoC number tints green, PV-now tints yellow). Financial/state colours (--ok/--warn/--bad) appear only on signed deltas, pending/attention markers, and status dots. Tariff-band colours appear only where the band IS the meaning (timeline cells, rate bars, legends — tint the VALUE or a leading dot, never flood a surface). The single interaction accent (--accent blue) is reserved for focus, active nav, links, the live-pulse, and the segmented-control thumb — nothing static is blue for energy. If removing a colour loses no meaning, it is removed. ICONS: one thin-line geometric family on a 24×24 viewBox, 1.75px stroke (1.5px only at ≤14px), round caps/joins, vector-effect:non-scaling-stroke, currentColor only — they default to --text-dim in headers, --text when active, and NEVER carry their own colour (the lone permitted exceptions are inside the power-flow nodes and the Efficiency self-use banner, where the icon IS its domain). Sizes: 18px in widget headers, 16px inline, 20px in power-flow/legend/nav, 14px for sub-labels (1.5px stroke). Every emoji and stray text glyph (⚡💰📤♨📊📈🔋☀🌙🌗 ↺ → ✓ ↓ ↑ •) is replaced. TYPOGRAPHY: dramatic size/weight contrast carries hierarchy, not colour. Scale (final): --font-2xs 0.68rem/600, --font-xs 0.75rem/500, --font-sm 0.875rem/500, --font-md 1rem/400, --font-lg 1.125rem/600, --font-xl 1.5rem/600, --font-2xl 2rem/700, --font-display 3.25rem/700/lh1.0/-0.02em (the focal number per card), --font-hero 4.5rem/700/lh0.95/-0.03em (the one anchor number for the home screen). Every numeral uses font-variant-numeric: tabular-nums. Units render at ~0.5em/500/--text-dim. Widget header labels stay UPPERCASE 0.7rem/700/0.12em. System font stack only (no webfont). MOTION: two classes only. (1) ENTRANCE — runs once on mount, choreographed: every .widget rises opacity 0→1 + translateY(12px)→0 + scale(0.985)→1 over 420ms (--dur-enter) with --ease-entrance cubic-bezier(0.22,1,0.36,1), staggered 40ms (--stagger-step) per card capped at 280ms; the surface's focal number animates LAST, ~300ms after the grid settles, scaling 0.96→1 over 500ms with --ease-lock cubic-bezier(0.34,1.3,0.64,1) for a ~2% overshoot lock-in, while useAnimatedNumber counts up. (2) LIVE — continuous, reserved exclusively for energy-flow particles (speed/density/radius map to kW) and the single live-pulse dot (1600ms ease-in-out, OPACITY-ONLY 0.4→1→0.4 infinite, only beside genuinely-live readouts). Static cards never loop/breathe/shimmer. Hover is a depth lift (translateY(-2px), --shadow→--shadow-lg, 160ms ease-out), never a colour change. Theme switch cross-fades 220ms. CRITICAL: all entrance animates transform/opacity only over a pre-sized grid (fixed grid-template + min-heights on every widget) so nothing reflows on load. Under prefers-reduced-motion: entrance/stagger/lock-in/dash-draw/chart morph disabled (final state instant), useAnimatedNumber snaps, particles freeze into static proportional-width connector lines with kW labels, live-pulse becomes a static dot, body::before drift pauses, ECharts animation:false; hover lift kept but capped to 1px.

## Typography scale

One focal number per card, and it is the ONLY thing at display size — everything supporting it (label, unit, delta, source) sits at --font-sm or smaller in --text-dim/--text-mute. Hierarchy is carried by SIZE and WEIGHT contrast, not colour or borders: focal numbers jump 3-4 steps above their label so the eye lands instantly (Apple's dramatic-contrast rule). All numerals use font-variant-numeric: tabular-nums + a fixed feature so digits don't shimmer as live values tick (pairs with useAnimatedNumber). Units render at ~0.5em of the number, weight 500, --text-dim, with a hair of letter-spacing so '4.2 kW' reads as one object with the number dominant. Widget header labels stay UPPERCASE 0.7rem 700 tracked 0.12em (kept from current) — they're chrome, deliberately tiny. Line-height tightens as size grows: 1.05 for display, 1.5 for body. Use -apple-system/system-ui stack already in base.css; do NOT add a webfont (keeps it Apple-native and avoids FOUT on a live dashboard).

- `--font-2xs 0.68rem / 600 — micro labels, footer, chip text`
- `--font-xs 0.75rem / 500 — badges, deltas, source tags, axis labels`
- `--font-sm 0.875rem / 500 — secondary values, list rows, nav tabs`
- `--font-md 1rem / 400 — body copy, default`
- `--font-lg 1.125rem / 600 — sub-headings, secondary card numbers`
- `--font-xl 1.5rem / 600 — section titles (h2)`
- `--font-2xl 2rem / 700 — page hero title (h1)`
- `--font-display 3.25rem / 700 / line-height 1.0 / tabular-nums / letter-spacing -0.02em — THE focal number per card (kW now / SoC% / £ today)`
- `--font-hero 4.5rem / 700 / line-height 0.95 / letter-spacing -0.03em — the single home-screen anchor number (the one focal point of the whole dashboard, e.g. live net power in the energy-flow hero)`

## Colour discipline

Monochrome-first. The default state of every card is greyscale: --bg-card surface, --text/--text-dim/--text-mute for all text, NO accent edge, NO corner tint. Colour is spent only to encode a real-world physical or financial state — it is information, never decoration. The ONE interaction accent (blue) is reserved for focus rings, active nav, links, and live-data pulse — nothing static is blue just to look energetic. This directly kills the named weaknesses: per-widget accent edges and corner tints are DEPRECATED outright; the 7-tone Widget system collapses to monochrome + semantic inline colour only.

**Semantic use:** ENERGY DOMAINS (used ONLY on the live power-flow particles, domain charts, and the single value that represents that domain — e.g. the SoC number can tint green, the PV-now number can tint yellow): --pv #fbbf24 = solar generation; --batt/--export #10b981 = battery charge + energy exported (green = energy you own/keep/earn); --grid #60a5fa = grid/mains direction-neutral; --house #c084fc = household load; --import #ef4444 = energy bought from grid (red = money leaving). FINANCIAL/STATE (used on deltas, badges, status dots): --ok #10b981 = saving / good / on-plan / positive delta; --warn #f59e0b = caution / peak window / attention; --bad #ef4444 = cost / overspend / error / negative delta. TARIFF BANDS (timeline + tariff strip ONLY, where the band IS the meaning): --neg-price blue, --cheap green, --standard grey, --peak amber, --peak-export red. INTERACTION: --accent #3b82f6 = focus, active tab, links, live pulse — the single Tesla accent, used sparingly. Rule of thumb: if removing the colour loses no meaning, remove it.

**Deprecate:** widget.css .widget-accent (the 3px left edge bar) — delete the element + all --widget-accent declarations; domains read via the header icon + focal-number tint instead; widget.css --widget-tint corner gradient (the linear-gradient(160deg,...) in .widget background) — delete; cards are flat --bg-card with depth from shadow only; the WidgetTone union's colour mapping (power/tariff/thermal/savings/plan/coming) — keep the prop name for one release for source-compat but map ALL tones to the same monochrome card; tone no longer paints anything; widget-header-icon filter: saturate(1.1) — remove; icons are currentColor monochrome now; ThemeToggle emoji (🌙☀️🌗) — replace with the single 'theme' line icon whose fill encodes mode; every emoji passed as a Widget `icon` prop across surfaces — replace with the thin-line icon set; ::selection hard-coded rgba(59,130,246,0.4) is fine to keep but should reference the accent; body::before three-colour radial wash — keep but soften (see tokenChanges); it currently sprays pv-yellow + batt-green + accent-blue which competes with the meaning-only rule. Reduce to a single near-neutral accent-tinted glow so colour stays meaningful elsewhere.

## Motion

- **Principles:** Meaningful motion only (Apple). Two motion classes: (1) ENTRANCE — runs once on mount, choreographed; (2) LIVE — continuous, reserved exclusively for the energy-flow particles and the live-pulse dot, because only live data should visibly move. Static cards do NOT loop, breathe, or shimmer. Hover is a subtle depth lift, not a colour change. All springs are critically-damped-ish (no bouncy overshoot except a tiny 1-2% on the hero number settle). Transforms + opacity only (GPU-cheap, never animate layout/box-shadow width). Numbers count up via the existing useAnimatedNumber hook on first load and on value change.
- **Entrance:** Staggered card rise on the home/plan grids. Each .widget enters with opacity 0→1 and transform translateY(12px)→0, scale(0.985)→1. Stagger: card N delay = 40ms * N, capped at 280ms total (so an 8-card grid finishes in <320ms and never feels slow). Duration per card 420ms, easing cubic-bezier(0.22, 1, 0.36, 1) (an ease-out 'expo-ish' — fast start, soft settle, no bounce). The hero focal number animates LAST and slightly bigger: after the grid settles (~300ms), the hero number scales 0.96→1 over 500ms with cubic-bezier(0.34,1.3,0.64,1) for a ~2% overshoot 'lock-in'. CRITICAL: reserve final layout up-front (min-heights on widgets, fixed grid template) so nothing reflows/jumps as cards arrive — the named 'layout jumps on load' weakness is fixed by animating only transform/opacity over a pre-sized grid.
- **Timings:** Entrance card: 420ms / cubic-bezier(0.22,1,0.36,1), stagger 40ms (cap 280ms). Hero number lock-in: 500ms / cubic-bezier(0.34,1.3,0.64,1). Hover lift: 160ms / ease-out, translateY(-2px) + shadow var(--shadow)→var(--shadow-lg). Nav/tab active: 140ms (keep existing 100ms→bump to 140ms for smoothness). Theme switch: 220ms ease on background-color + color (add a global transition so dark↔light cross-fades instead of snapping). Live particle traversal speed: maps to kW (faster = more power), NOT a fixed loop. Live-pulse dot: 1600ms ease-in-out opacity 0.4→1→0.4 infinite — the ONLY infinite animation allowed on a static element, and only next to genuinely-live readouts.
- **Reduced motion:** Under @media (prefers-reduced-motion: reduce): entrance stagger + rise + hero lock-in are disabled — cards appear at final state immediately (set animation: none, transform: none, opacity: 1). useAnimatedNumber snaps to final value (no count-up). The live energy-flow particles STOP traversing and render as static proportional-width connector lines with a static label (information preserved, motion removed). Live-pulse dot becomes a static filled dot. body::before drift animation pauses. Hover lift is kept (it's user-initiated and instantaneous) but capped to a 1px translate. Theme cross-fade is kept (short, non-vestibular). Provide this once at the shell level so every surface inherits it.

## Roadmap

### Phase 1 — Shared foundation (tokens, icons, motion primitives, hook). Everything depends on this; ship before any surface.

_These are the atoms every surface composes from. Locking the type scale, icon family (including the four shared extras), motion tokens, the de-chromed Widget, the reduced-motion behaviour, and the chart helpers FIRST guarantees the six surfaces stay coherent and prevents per-surface drift (the entire point of an Apple/Google design system). Nothing visual ships to users in this phase, but it de-risks all later work._

- Build the thin-line icon module (one file): all foundation icons (power-live, cost, export, import, heating, chart-bars, trend, battery with data-fill, solar, grid, schedule, settings, efficiency, theme) PLUS the four sanctioned additions (house, chevron, check, revert). 24×24, 1.75px (1.5px ≤14px), round caps/joins, vector-effect:non-scaling-stroke, currentColor. - widget.css: delete .widget-accent element + all --widget-accent declarations; remove --widget-tint corner gradient (background → flat var(--bg-card)); map all .widget--tone-* to no-op; remove widget-header-icon filter:saturate(1.1).
