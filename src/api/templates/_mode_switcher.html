System mode

Three independent toggles. Each change runs through the standard preview → modal → confirm flow.

Operation mode

—

simulation blocks ALL hardware writes from this service. operational allows the LP/dispatcher to push to Fox V3 (and Daikin if active mode).

Daikin control

—

passive = service NEVER writes to the heat pump (firmware autonomous). active = legacy v9 control (LP schedules tank temps, LWT offsets, etc.).

Require simulation-id for writes

—

When ON, every state-changing API call must first go through its /simulate endpoint and pass the simulation_id back as X-Simulation-Id. Protects against scripts and accidental writes — the cockpit always uses this flow regardless.
